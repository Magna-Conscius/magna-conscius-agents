# Website Representation Map

## Objective
Represent the full Magna Conscius organization on the website without turning the site into a raw mirror of internal folders.

The site should translate the internal structure into a public-facing system that is legible, credible, and strategically useful.

## Core Principle
The website should not expose internal folders directly.

It should translate them into public categories that communicate:
- what Magna Conscius is
- how it works
- what it produces
- why it matters

## Navigation Recommendation
Primary navigation should eventually become:
- About
- Methodology
- Research
- Blog
- Models
- Intelligence
- Services
- Products
- Contact

This is broader than the current nav, but it matches the organization much better.

## Content Discipline
Each public page should answer one clear question:

- About: what is Magna Conscius?
- Methodology: how does it work?
- Research: what is it studying?
- Blog: what is it thinking about publicly?
- Models: what does it try to predict?
- Intelligence: how does that become systems?
- Services: how can this be applied now?
- Products: what is being built?
- Contact: how do people engage?
